JOB TITLE ADJUSTMENT RULES (critical — read carefully):
The BASE RESUME may contain combined titles like "Software Engineer / ML Engineer / Data Analyst (Research Assistant)".
This is a raw data artifact — NEVER copy such combined titles into the tailored resume.
Always select ONE title that best reflects both (a) what the person actually did AND (b) the target role.

Decision process:
1. Look at the actual work done in that role (from the bullets)
2. Look at the target role title
3. Pick the single most appropriate title, keeping the original context in parentheses

Examples:
- Target: Software Engineer → "Software Engineer (Research Assistant)"
- Target: Data Scientist → "Data Scientist (Research Assistant)"
- Target: Backend Developer → "Backend Developer (Research Assistant)"
- Target: Fund Accountant → "Fund Accountant (Business Analyst)"
- Target: Business Analyst → "Business Analyst (Fund Accountant)"

The parenthetical preserves honesty for background checks. Never drop it entirely.

BULLET TAILORING (do this for every kept role):
- REORDER bullets: most JD-relevant accomplishments come first
- REPHRASE bullets: incorporate JD keywords where the underlying skill is genuine and directly relevant to the experience context
  Examples of good rephrasing:
  • JD says "HTTP APIs" + resume says "built REST APIs" → "Designed and implemented HTTP REST APIs"
  • JD says "microservices" + resume has microservice work → lead with that bullet
  • JD says "CI/CD" + resume mentions CI/CD → make it explicit and prominent
  • JD says "production systems" + resume says "pipeline" → "production data pipeline"
- Do NOT rephrase if the underlying experience doesn't support the keyword
